ISO Amp One-Click Login Implementation Guide
Complete Implementation Overview
The ISO Amp integration provides a secure, one-click login to their interchange cost calculator tool. Here's exactly how to recreate it:

Backend Implementation
1. Environment Variable Setup
Add the ISO Amp secret key to your environment:

ISO_AMP_SECRET_KEY=your_secret_key_here
2. Core Authentication Function
Create this exact function (requires Node.js crypto module):

import crypto from 'crypto';
function generateISOAmpLoginLink(
  email: string,
  name: string,
  destination: string = "/dustin"
): string {
  const secretKey = process.env.ISO_AMP_SECRET_KEY;
  if (!secretKey) {
    throw new Error("ISO_AMP_SECRET_KEY not found in environment variables");
  }
  const message = {
    email: email,
    name: name,
    destination: destination,
  };
  // JSON stringify and hex encode
  const messageJson = JSON.stringify(message);
  const hexEncodedMessage = Buffer.from(messageJson).toString("hex");
  // Get Unix timestamp
  const unixTime = Math.floor(Date.now() / 1000);
  // Create message blob
  const version = "1";
  const messageBlob = hexEncodedMessage + unixTime + version;
  // Generate HMAC signature
  const signature = crypto
    .createHmac("sha256", secretKey)
    .update(messageBlob)
    .digest("hex");
  // Build URL
  const baseUrl = "https://dmprocessing.isoquote.com/login_via_link";
  return `${baseUrl}?m=${hexEncodedMessage}&s=${signature}&t=${unixTime}&v=${version}`;
}
3. API Route (Express.js)
app.get("/api/iso-amp-quote", (req, res) => {
  try {
    const loginLink = generateISOAmpLoginLink(
      "dustin@dmprocessing.com",  // Replace with dynamic user email
      "Dustin Wilkins",           // Replace with dynamic user name
      "/dustin"                   // ISO Amp destination path
    );
    res.redirect(loginLink);
  } catch (error) {
    console.error("Error generating ISO Amp link:", error);
    res.status(500).send("Error connecting to ISO Amp");
  }
});

Key Implementation Details
Security Protocol
Message Construction: User details (email, name, destination) are JSON-encoded then hex-encoded
Timestamp: Unix timestamp prevents replay attacks
Version: Protocol version (currently "1")
Message Blob: Combined hex message + timestamp + version
HMAC Signature: SHA-256 HMAC using secret key ensures authenticity
URL Parameters:
m = hex-encoded message
s = HMAC signature
t = Unix timestamp
v = version
User Experience
Clicking the button opens ISO Amp in a new tab
User is automatically logged in (no manual credentials needed)
They can calculate precise interchange rates
User manually copies the rate back to your calculator
Integration Notes for Widget
User Info: You'll need to collect user email/name for the authentication
Simplified Version: For a sales widget, you could use hardcoded sales rep credentials
Environment: Make sure the ISO_AMP_SECRET_KEY is available in your widget's environment
Dependencies: Requires Node.js crypto module for HMAC generation
